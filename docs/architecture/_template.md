# [시스템 이름]

> 이 파일은 한 시스템의 설계를 Codex에게 넘기기 위한 문서다.
> 작성 순서: 사람+Claude가 설계 → 이 틀에 채움 → Codex가 읽고 구현.
> **빈 칸을 남기지 않는다.** 칸이 비면 그만큼 Codex가 추측하게 되고, 추측은 곧 잘못된 코드다.
> (작성 후 이 안내 인용문은 지워도 된다.)

---

## 1. 목적

- 이 시스템이 무엇을 하는가 (1~3줄)
- `project.md`의 어떤 증명 목표를 담당하는가 (예: "GAS — GE 기반 데미지 흐름")

## 2. 설계 의도 / 핵심 결정

> 이 항목이 가장 중요하다. **"왜 이렇게 했는가"** — 나중에 Velog·포트폴리오에 그대로 쓰일 부분이다.

- 핵심 결정과 그 근거 (예: "ASC Replication Mode는 Mixed로 한다. 이유는 …")
- 고려했지만 **택하지 않은 대안**과 그 이유 (탈락 사유가 포트폴리오에서 깊이를 보여준다)

## 3. 구조

- 핵심 클래스와 각자의 책임 (한 줄씩)
- 클래스 간 관계 / 소유 구조
- 필요하면 간단한 다이어그램이나 텍스트 트리

## 4. 데이터 / 실행 흐름

- 입력부터 결과까지 한 흐름을 순서대로 (예: 입력 → 어빌리티 활성 → GE 적용 → Attribute 변경 → 반영)

## 5. 네트워크 측면 (해당 시 필수)

> 이 프로젝트는 Dedicated Server가 전제다. 거의 모든 시스템이 이 항목을 채워야 한다.

- 무엇이 **서버 권위**인가
- 무엇을 복제하는가 / 복제하지 않는가
- 복제 조건 (`ReplicationCondition` / `COND_` 사용 여부와 이유)
- 클라이언트 예측(prediction) 유무
- 최적화 고려 (Relevancy, Dormancy, Lag Compensation 등 해당 시)

## 6. 시스템 고유 제약

- 전역 `constraints.md` 외에, 이 시스템에서만 지켜야 할 규칙
- 자주 실수할 만한 지점에 대한 경고

## 7. 구현 지시 (Codex용) — 페이즈 단위

> 이 시스템을 **사람이 검증 가능한 페이즈(P0, P1, …)로 쪼개** 순서대로 넘긴다.
> 페이즈 경계는 *구현 편의*가 아니라 **"사람이 눈으로 확인할 수 있는 동작이 처음 나오는 지점"**으로 긋는다.
> (예: "AttributeSet만 정의"는 페이즈가 아니다 — 확인할 게 없다. "쏘면 서버에서 Health가 깎이고 클라에 반영된다"가 페이즈다.)
>
> **규칙:**
> - Codex는 각 페이즈 끝의 멈춤 지점에서 **반드시 멈추고 대기**한다. 다음 페이즈로 스스로 넘어가지 않는다.
> - 검증은 **사람이 한다.** Codex의 "동작합니다" 자가 보고는 통과 근거가 아니다.
> - 사람이 통과 판정 → 커밋 → "다음" 지시가 있을 때만 다음 페이즈 진행. (이 커밋 지점이 곧 세션 재개 지점이다.)
> - 페이즈 수는 **검증 가능 지점의 최소 개수**로. 잘게 쪼갤수록 사람 검증 부담이 그만큼 늘어난다. seam이 알려주는 만큼만 나눈다.

### P0 — [이 페이즈가 끝나면 동작하는 것, 한 줄]
- **만들 것 (순서):** 1) … → 2) … → 3) …
- **멈춤 지점:** 여기까지 하고 멈춰 보고한다.
- **통과 기준 (사람 확인):** 컴파일 통과 + [무엇을 어떻게 확인하면 통과인지 구체적으로]

### P1 — […]
- **만들 것 (순서):** …
- **멈춤 지점:** …
- **통과 기준 (사람 확인):** …

(필요한 만큼 P2, P3 … 반복. 쪼갤 필요 없으면 P0 하나로 끝나도 된다.)

## 8. 시스템 인수 검증 (전체)

> 7항이 페이즈별 *국소* 검증이라면, 여기는 **이 시스템이 통째로 `project.md`의 증명 목표를 만족하는지** 보는 최종 시나리오다.

- 전체 동작 시나리오 (입력부터 결과까지 한 번에)
- 네트워크 시스템이면 **서버-클라 검증 방법** 명시 (예: Dedicated Server + 클라 2개 PIE → … 확인)
- 이 시스템이 담당하기로 한 증명 목표(2항 · `project.md`)가 실제로 시연되는가

- (비어 있으면 "없음"이라고 적는다 — 빈 칸을 남기지 않기 위해)
